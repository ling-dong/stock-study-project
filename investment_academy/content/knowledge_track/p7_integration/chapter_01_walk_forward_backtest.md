# 第一章：Walk-Forward 回测

## 1.1 为什么需要 Walk-Forward？

常规回测有一个致命问题：你在 2026 年回测 2020-2025 的数据，意味着你用了**未来视角**来调参数——即使你主观上没有，客观上你看到全部数据后调整参数就是在过拟合。

**Walk-Forward 回测**就是为了解决这个问题。

## 1.2 Walk-Forward 的工作流程

```
┌─────────────────────────────────────────────────────────┐
│ 完整数据: 2020-01 ~ 2026-06                             │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  训练窗口(6个月)    测试窗口(1个月)                      │
│  ┌──────┐          ┌──┐                                 │
│  │2020H1│  →优化→  │7月│  → 记录测试结果                 │
│  └──────┘          └──┘                                 │
│         ┌──────┐          ┌──┐                          │
│         │2020H2│  →优化→  │1月│  → 记录测试结果          │
│         └──────┘          └──┘                          │
│                ...滚动前进...                            │
│                                                         │
│  最终：汇总所有"测试窗口"的收益 → 这才是真实回测结果      │
└─────────────────────────────────────────────────────────┘
```

### 关键原理

- **训练窗口**：用 6 个月历史数据训练模型/调优参数
- **测试窗口**：用下一个月的数据检验，这一个月你从未看过
- **滚动**：窗口向前滚动 1 个月，重复
- **最终结论**：把所有测试窗口的收益汇总

> 🔑 Walk-Forward 的核心：**模型调优和模型检验的数据永不重叠。** 训练用的数据绝对不会出现在测试集中。

## 1.3 SPAS 的 Walk-Forward 实现

SPAS 的 `engine.py` 使用 **6+1 窗口**：

```
训练窗口 = 6 个月（约 120 个交易日）
测试窗口 = 1 个月（约 20 个交易日）
滚动步长 = 1 个月
```

每滚动一次：
1. 用 6 个月数据重新训练 ML 模型
2. 用 6 个月数据重新计算规则引擎的参数
3. 在接下来的 1 个月测试窗口内执行交易
4. 记录该月的收益率、波动率、最大回撤

输出：
```
测试窗口     训练数据        收益率      夏普    最大回撤
2023-07      2023-01~06     +2.3%      1.2     -3.1%
2023-08      2023-02~07     -1.1%     -0.5     -4.8%
...
2026-06      2026-01~06     +4.1%      1.8     -2.2%
────────────────────────────────────────────────
汇总均值: +1.8%/月    夏普均值: 0.9    最大回撤: -7.2%
```

## 1.4 如何解读 Walk-Forward 结果

| 指标 | 优秀 | 合格 | 不合格 |
|------|------|------|--------|
| 月度均收益 | >2% | >1% | <0.5% |
| 夏普比率 | >1.2 | >0.8 | <0.5 |
| 月胜率 | >65% | >55% | <50% |
| 最大回撤 | <10% | <20% | >25% |
| 连续亏损月数 | 1-2 | 2-3 | >4 |

> ⚠️ Walk-Forward 表现和实盘表现的差距就是你策略的"过拟合程度"。差距越小，策略越可靠。

## 1.5 Walk-Forward 的局限

1. **数据量不足**：A 股很多 ETF 只有 3-5 年数据，能做约 30 个滚动窗口，结论的统计显著性有限
2. **市场结构性变化**：政策、规则、参与者结构的变化可能让历史关系不再成立
3. **成本低估**：回测中交易成本的估算往往低于实盘

## 本章要点

| 概念 | 说明 |
|------|------|
| Walk-Forward | 训练和测试永不重叠的回测方法 |
| 6+1 窗口 | SPAS 用 6 月训练 + 1 月测试 |
| 滚动步长 | 每次向前滚动 1 个月 |
| 过拟合程度 | WF 结果和实盘的差距 |
