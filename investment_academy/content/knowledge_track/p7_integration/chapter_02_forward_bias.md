# 第二章：前视偏差

## 2.1 什么是前视偏差？

> **前视偏差**是回测中最隐蔽也最危险的错误——你在"今天"使用了"未来"的信息。

这不是故意的作弊，而是在数据处理中不经意间犯了逻辑错误。

## 2.2 常见的前视偏差类型

### 类型一：数据对齐错误

```
错误：今天的交易信号用今天的收盘价来计算
      然后在"今天开盘"入场
      
问题：今天收盘价在交易时还不知道！
      → 回测收益被高估

正确：用昨天的数据计算信号
      在今天开盘入场
      用今天收盘价计算盈亏
```

### 类型二：幸存者偏差

```
错误：回测时用的 ETF 列表是"现在还存在的 ETF"
      
问题：历史上可能有 ETF 退市/清盘，这些"消失的 ETF"没在回测里
      → 回测收益被高估

正确：回测时用"历史上每个时点实际存在的 ETF 列表"
```

### 类型三：停牌/涨跌停处理

```
错误：回测中没有处理停牌和涨跌停
      → 假设所有交易日都能自由买卖

问题：A 股涨跌停时几乎无法交易
      → 回测收益被高估

正确：标记停牌日和涨跌停日，这些日期不产生交易信号
```

## 2.3 SPAS 的前视偏差检测

SPAS 的 `forward_bias.py` 对每条交易记录做自动化检查：

| 检查项 | 方法 |
|--------|------|
| 数据时点 | 确认信号计算使用的所有数据都在信号触发时点之前 |
| 股价调整 | 确认分红/送股后的价格已经正确调整 |
| 涨跌停过滤 | 确认交易日在涨跌停状态下没有"执行"交易 |

## 2.4 如何自己避免前视偏差

1. **数据排序**：确保所有数据按照时间顺序插入，正序访问
2. **时点标记**：每个计算值记录它被计算的时间点
3. **回溯检查**：写完回测代码后，随机抽查 10 条交易记录，逐条验证

## 本章要点

| 偏差 | 说明 |
|------|------|
| 数据对齐 | 今天收盘价不能用于今天开盘决策 |
| 幸存者偏差 | 用历史上存在的 ETF 列表 |
| 涨跌停 | 不可交易的日期不能产生"虚拟"收益 |
| 时点标记 | 每个计算值记录计算时点 |
