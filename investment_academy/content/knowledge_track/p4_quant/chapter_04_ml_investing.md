# 第四章：ML 在投资中的应用

## 4.1 机器学习能做什么和不能做什么

### ✅ 能做好的

| 任务 | 成熟度 | 说明 |
|------|--------|------|
| 趋势分类 | 高 | 判断上涨/下跌/震荡 |
| 涨跌概率预测 | 中高 | 输出"明天涨的概率是 65%" |
| 异常检测 | 高 | 识别异常放量、异常波动 |
| 模式识别 | 中 | 自动发现 K 线形态 |

### ❌ 做不好的

| 任务 | 原因 |
|------|------|
| 精准价格预测 | 股价是随机游走 + 增量信息驱动 |
| 预测黑天鹅 | ML 只学历史，预测不了历史上没发生过的事 |
| 长期择时 | 信号噪音比随预测周期指数下降 |

> 💡 ML 在投资中的正确角色：**提高概率判断的精度**，而非预测明天的收盘价。

## 4.2 SPAS 使用的模型

### LightGBM（梯度提升树）

处理表格数据的最佳选择之一。优势：
- 速度快，可处理大量特征
- 天然处理缺失值和非线性关系
- 特征重要性输出清晰

SPAS 使用 `GBClassifier` 对趋势方向做三分类（涨/跌/震荡）。

### 概率校准（Isotonic Calibration）

ML 模型输出的"0.8 概率"不一定真的是 80%。模型可能过于自信（输出 0.9 实际只有 0.7）或过于保守。

**概率校准**将模型输出的原始分数映射到真实概率：

```
未校准输出：score = 0.75 → "75% 概率上涨"
校准后发现：历史上所有 score=0.75 的信号，实际只有 60% 是真的涨了
校准后输出：probability = 0.60 → "60% 概率上涨"
```

SPAS 使用 Isotonic Regression 做概率校准，ECE（Expected Calibration Error）目标是 <0.05。

> 🔑 **未经校准的概率输出是危险的。** "85% 概率上涨"和"60% 概率上涨"对应完全不同的仓位决策。

## 4.3 ML 模型在实战中的局限

### 数据外的世界

所有 ML 模型都基于历史数据训练。当市场发生结构性变化时（新政策、新技术突破、黑天鹅），模型的预测力会急剧下降。

### 曲棍球效应

ML 模型在训练数据上的表现 ≠ 实盘表现。典型情况：
```
训练集样本内：准确率 85%
验证集：准确率 72%
实盘：准确率 58%
```

这个衰减是普遍规律。不要因为训练集表现好就盲目乐观。

### 市场的适应性

市场本身会学习。当一个 ML 信号被广泛使用时，它的预测力会下降（因为太多人在它触发时交易，提前消化了信号）。

## 4.4 融合：规则 + ML

纯粹的 ML black-box 策略不可靠，纯粹的规则策略又难以捕捉复杂模式。最佳做法是融合：

```
规则引擎                      机器学习
  │                             │
  │ 生成 8 维规则向量          │ 生成概率预测
  │ 基率收缩估计               │ 概率校准
  │                             │
  └──────────┬────────────────┘
             ▼
         融合层
     动态权重投票
             │
             ▼
         最终决策
```

规则提供**可解释性 + 风控底线**，ML 提供**超出规则的模式识别能力**。

## 4.5 Brier 分数 — 评估概率预测的质量

Brier 分数衡量概率预测的准确度：

```
Brier = Σ(预测概率 - 实际结果)² / n
Brier < 0.20：可用（金融领域）
Brier < 0.15：优秀
Brier < 0.10：非常优秀（金融领域极少达到）
```

> ⚠️ 金融预测的 Brier 基准与气候/竞赛不同。随机猜测的基线约 0.25，量化策略能做到 0.18-0.22 已属出色。

SPAS 持续跟踪每个模型的 Brier 分数，Brier 恶化超过阈值时自动降低该模型的权重。

## 本章要点

| 概念 | 说明 |
|------|------|
| ML 能做 | 概率预测、趋势分类、异常检测 |
| ML 不能做 | 精准价格预测、黑天鹅预测 |
| 概率校准 | 将模型分数映射到真实概率 |
| ECE | 期望校准误差，衡量校准质量 |
| Brier 分数 | 概率预测的准确度评分 |
| 规则+ML | 最佳实践：融合而非替代 |
