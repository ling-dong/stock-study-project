# 第三章：特征工程

## 3.1 什么是特征？

**特征**是将原始数据（OHLCV）转化为可被模型理解的"描述语言"。原始的价格数字本身（3.52 元）没有投资含义，但"实体比 = 0.45"就有含义。

## 3.2 特征分类

| 类别 | 例子 | 计算复杂度 |
|------|------|-----------|
| **价格特征** | 涨跌幅、实体比、收盘位置 | 低 |
| **量价特征** | 量比、换手率、OBV | 中 |
| **趋势特征** | MA 斜率、ADX | 中 |
| **波动率特征** | 振幅、ATR、历史波动率 | 中 |
| **形态特征** | H2 得分、FB 得分 | 高 |
| **交叉特征** | 涨跌幅 × 量比、实体比 × ADX | 自定义 |

## 3.3 特征的时点标注（Point-in-Time）

这是量化投资中最容易被忽略但最重要的问题：**你在什么时候"知道"这个特征？**

```
错误：用收盘价计算信号，然后用当天开盘价入场
  → 前视偏差（Future Leak）
  
正确：用上个交易日的特征，判断今天开盘时是否入场
  → 信息在可用时点才被使用
```

SPAS 的 `bar_feature.py` 对每个特征都记录了计算时点，确保回测中没有未来信息泄露。

## 3.4 SPAS 的特征向量

SPAS 对每根 K 线生成一个 **FeatureVector**，包含 50+ 维特征：

```
FeatureVector = {
    body_ratio, close_pos, upper_shadow, lower_shadow,  # K线6维
    trend_bar, limit_status,
    ema5_slope, ema20_slope,                             # 趋势特征
    volume_ratio, turnover_anomaly,                       # 量价特征
    h2_score, l2_score, fb_score,                         # 形态特征
    sector_momentum, market_heat,                          # 板块特征
    ...
}
```

## 3.5 特征选择

不是特征越多越好。太多特征会导致"维度灾难"——样本看起来变稀疏了。

常用方法：
1. **相关性分析**：去掉与其他特征高度相关（>0.85）的冗余特征
2. **重要性排序**：用模型的特征重要性排序，砍掉排名后 50%
3. **业务判断**：去掉没有投资含义的纯统计特征
