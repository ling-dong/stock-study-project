# 第五章：融合决策

## 5.1 单一信号的问题

任何单一信号都有盲点：

| 信号来源 | 盲点 |
|----------|------|
| 规则策略 | 无法捕捉复杂非线性模式 |
| ML 模型 | 黑箱，不知为何做出判断 |
| 技术指标 | 每个指标都有失效场景 |
| 多时间框架 | 不同框架可能给出相反信号 |

## 5.2 融合的两种模式

### 模式一：投票融合

每个信号源（规则、ML、各时间框架）各自投票，加权汇总：

```
总信号 = w₁×规则 + w₂×ML + w₃×日线 + w₄×60分 + w₅×板块
```

| 总信号 | 决策 |
|--------|------|
| >0.3 | 做多 |
| -0.3 ~ 0.3 | 观望 |
| <-0.3 | 做空/减仓 |

### 模式二：分歧处理

当信号源之间有大的分歧时：

```
规则说"买"，ML 说"卖"
  → 不操作，等待分歧缩小
  → 分歧大 = 市场方向不明 = 交易胜率骤降
```

> 🔑 **分歧不是加仓或对冲的理由，而是观望的理由。**

## 5.3 SPAS 的动态权重

SPAS 的 Fusion 层使用 **sigmoid 动态权重**：

```
w_i = sigmoid(accuracy_i_recent × ADX_i × volatility_adjustment)

accuracy_i_recent：该信号源近期的准确率
ADX_i：该时间框架的趋势强度
volatility_adjustment：波动率调整因子
```

- 近期表现好的信号源 → 权重自动上升
- 趋势强的框架 → 权重自动上升
- 高波动率时 → 所有信号源的权重都被压低

## 5.4 融合决策的 Checklist

```
入场前确认：
□ 融合总信号 > 0.3（做多）或 < -0.3（做空）？
□ 信号源之间的分歧 < 0.5？
□ 至少 2 个时间框架信号方向一致？
□ 板块热度不处于极端（0.2 < 消耗指数 < 0.8）？

全部满足 → 按计划仓位入场
任何一项不满足 → 观望
```

## 本章要点

| 概念 | 说明 |
|------|------|
| 投票融合 | 多信号源加权投票 |
| 分歧处理 | 分歧 = 方向不明 = 观望 |
| 动态权重 | 近期表现好 → 权重高 |
| sigmoid | SPAS 的权重产出函数 |
